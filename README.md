# Spot the Fake Photo

Given one image, decide if it's a **real photo** or a **photo of a screen** (someone re-photographing a phone or laptop instead of the real thing).

Salescode.ai take-home submission.

**Live demo:** https://salescode-spot-the-fake.onrender.com — works on desktop (drop an image) and mobile (live camera capture).
*Note: hosted on Render's free tier, which spins down after 15 min of inactivity. First request after idle takes ~30–60s; subsequent calls are ~100 ms.*

## Results

- **Honest cross-device accuracy:** 84.7% mean across leave-one-screen-out folds (range 81.4–88.9%).
- **Recall on unseen screens:** 92% mean (HP 92%, Mac 89%, OnePlus 95%).
- **Latency:** ~105 ms/image, MacBook M2 Pro CPU.
- **Cost:** free on-device, ~$0.50/million if served on a small cloud CPU.

Full write-up: **[NOTE.md](NOTE.md)**.

## How it works

7 hand-crafted features per image (residual-domain moiré peaks, broadband HF energy, horizontal banding, high-pass residual texture stats) → standardized → SVM-RBF with calibrated probabilities. No neural network, no GPU.

## Files

| File | What it does |
| --- | --- |
| `features.py` | Extracts the 7 features from an image. Single source of truth for training and inference. |
| `train.py` | Loads `dataset/features.csv`, runs 5-fold CV + leave-one-screen-out, fits the SVM, saves `model.pkl`. |
| `predict.py` | `python predict.py image.jpg` → prints a score in `[0, 1]`. |
| `app.py` + `templates/index.html` | Flask web demo: upload or camera capture, returns a live score. |
| `build_dataset.ipynb` | Builds the dataset and `features.csv`, plus the sanity-check analysis behind every claim in `NOTE.md`. |
| `model.pkl` | Trained SVM-RBF + feature list, generated by `train.py`. |
| `NOTE.md` | The submission write-up: approach, accuracy, trade-offs, what I'd improve. |

## Setup

```bash
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

## Train

```bash
python train.py
```

Prints CV + leave-one-screen-out numbers, writes `model.pkl`.

## Predict

```bash
python predict.py path/to/image.jpg
```

Prints one number: `0` = real, `1` = screen.

## Run the web demo locally

```bash
python app.py
```

Open `http://localhost:5050` in your browser.
